# What Is a SQL Query?

A SQL query is a statement sent to a database requesting information or instructing the database to perform an operation.

Examples include:

- Retrieve customer information.
- Insert a new transaction.
- Update account balances.
- Delete inactive records.
- Create new tables.

Although the word "query" often refers to retrieving data, in practice developers use "query" to describe almost any SQL statement.

# Anatomy of a SQL Query

Consider this example:

```sql
SELECT customer_name
FROM customers
WHERE customer_id = 101;
```

The query has three main parts:

```
SELECT

↓

What data do I want?

↓

FROM

↓

Where is the data?

↓

WHERE

↓

Which rows should be returned?
```

As we progress through the book, you'll learn every clause in detail.

# Characteristics of Good SQL

Good SQL should be:

- Correct
- Readable
- Efficient
- Maintainable
- Consistent
- Secure

Good SQL is not just about getting the correct result—it should also be easy for others to understand and maintain.

---

# Readability Matters

Compare the following examples.

Poor:

```sql
select*from customers where customer_id=101;
```

Better:

```sql
SELECT
    customer_id,
    customer_name
FROM customers
WHERE customer_id = 101;
```

Both queries produce the same result.

The second query is much easier to read and maintain.

# Ad Hoc vs Production Queries

## Ad Hoc Query

Written to answer an immediate question.

Example:

> How many customers registered today?

These queries are often temporary.

---

## Production Query

Used repeatedly by applications, reports, ETL pipelines, or dashboards.

Production queries should be:

- Tested
- Optimized
- Reviewed
- Documented

---

# Interactive Queries vs Application Queries

## Interactive

Written manually in tools like:

- SQL Server Management Studio
- DBeaver
- Azure Data Studio
- Snowflake Worksheets

---

## Application Queries

Generated by:

- Banking applications
- Mobile apps
- APIs
- ETL pipelines
- BI tools

These often execute thousands or millions of times each day.

# 🧠 Engineering Insight

Professional engineers spend more time **reading SQL** than writing SQL.

A well-formatted query can save hours of debugging and maintenance.

Always write SQL for humans first and computers second.

---

# Deep Dive

A SQL query is more than just text.

Internally, the database converts your SQL into execution plans, physical operators, and storage operations.

This is why two different SQL statements can produce the same result but have dramatically different performance.
